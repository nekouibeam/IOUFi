🔄 檔案間的互動與角色關係

1. 內部依賴關係 (Backend Logic)
工廠與實例：FundraiserFactory.sol 內部引用了 Fundraiser.sol 。
當使用者呼叫 createFundraiser() 時，工廠會執行 new Fundraiser(...) 並將呼叫者 (msg.sender) 設為該新專案的 owner 。
標準庫引用：Fundraiser.sol 繼承 Ownable ，確保只有該募資專案的建立者可以執行 withdraw() 提款動作 。

2. 前後端連動橋樑 (The Bridge)
ABI 供應：前端 React 需要知道合約有哪些函數。這由 out/ 中的 .json 檔案提供 。
地址對接：前端需要知道合約部署在哪。這由 broadcast/ 紀錄的地址提供 。
自動化流程：go.cjs 腳本扮演搬運工，將上述兩項資訊從 Solidity 目錄打包，傳遞到前端目錄供 ethers.js 初始化合約物件 。

3. 完整操作流 (Workflow)
開發：在 src/ 編寫合約。
編譯：forge build 產生 out/ 的 ABI 。
部署：script/ 執行部署並在 broadcast/ 紀錄地址 。
同步：執行 go.cjs 同步合約資訊至前端 。
互動：使用者透過 React 介面觸發 Factory 產生新募資案，或對特定的 Fundraiser 捐款 。

這套架構確保了後端邏輯 (Solidity)、部署流程 (Foundry) 與前端介面 (React) 之間既能獨立運作，又能透過自動化腳本緊密結合。
